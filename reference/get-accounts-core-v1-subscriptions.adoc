= 列出所有订阅
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/subscriptions`#

返回集合中所有匹配资源的 JSON 数组。如果在请求中指定了单个字段，则项目 JSON 数组将包含按指定顺序为每个匹配资源请求的字段。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| 继续 |  | 查询 | False  a| 
指示要从中恢复列表的令牌。

URL 示例："continue=+++<continue token="" from="" collection="" metadata="">+++"+++</continue>+++

OpenAPI 示例："+++<continue token="" from="" collection="" metadata="">+++</continue>+++</continue>



| count | string | 查询 | False  a| 
指示是否应在元数据对象中返回资源总数。

URL 示例："count=true"

OpenAPI 示例："true"



| 筛选器 | string | 查询 | False  a| 
指示在列出集合时应匹配哪些字段以返回资源。

支持的运算符有：

* eq - 相等
* lt/gt - 小于/大于
* lte/gte - 小于或等于/大于或等于


URL 示例："filter=field%20eq%20%27value%27"

OpenAPI 示例："field eq 'value'"



| 包括 | string | 查询 | False  a| 
指示列出集合时应返回哪些字段。

URL 示例："include=id"、"include=id,name"

OpenAPI 示例："id"



| limit | string | 查询 | False  a| 
指示列出集合时要返回的最大资源数。

URL 示例："limit=2"

OpenAPI 示例："2"



| 订单 | string | 查询 | False  a| 
指示列出集合时资源的排序顺序。

URL 示例："orderBy=name"，"orderBy=name%20desc"

OpenAPI 示例："name desc"



| 跳过 | string | 查询 | False  a| 
指示列出集合时要跳过的资源数。

URL 示例："skip=1"

OpenAPI 示例："1"

|===

|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-subscription"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"
* "1.2"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| customerProfileID | string | True  a| 
未为试用期设置包含来自 http://authorize.net["Authorize.net"]的客户配置文件 ID 的 JSON 字符串，该字符串由 UI 调用。如果未设置，则在读取操作期间返回空字符串。定义的值为：

* 最小长度：0
* 最大长度：63




| paymentProfileID | string | False  a| 
包含由用户界面调用的付款账户 ID http://Authorize.net["Authorize.net"] 的 JSON 字符串，未设置为试用期。如果未设置，则在读取操作期间返回空字符串。定义的值为：

* 最小长度：0
* 最大长度：63




| paymentExpiry | string | False  a| 
JSON 字符串，包含指示付款方式何时过期的时间戳。这是从 authorize.net](http://authorize.net[]) 获得的，由 UI 设置。定义的值为：

* 符合 ISO-8601 日期时间架构。此字段不会为试用条款返回。




| purchaseOrderNumber | string | False  a| 
包含付款方式的采购订单编号的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：31




| 市场 | string | False  a| 
JSON 字符串，表示将记录付款的账户的市场。定义值：

* "netapp"
* "azure"
* "aws"
* "gcp"




| licenseSN | string | False  a| 
包含付款方式的许可证序列号的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：31




| 条款 | string | True  a| 
表示帐户当前层的 JSON 字符串。定义的值为：

* "试用"
* "付费"




| 状态 | string | True  a| 
表示订阅当前状态的 JSON 字符串。定义的值为：

* "活动"
* "非活动"




| appLimit | 数字 | True  a| 
表示此订阅的应用程序限制的 JSON 编号。值 -1 表示无限制。



| namespaceLimit | 数字 | True  a| 
表示此订阅的命名空间限制的 JSON 编号。值 -1 表示无限制。



| subscriptionPeriod | 数字 | True  a| 
表示订阅期限的 JSON 数字。值 -1 表示无限制。单位：天



| gracePeriod | 数字 | True  a| 
JSON 数字，表示试用订阅过期或付费订阅被取消后的宽限期。单位：天



| reminderBeforePeriod | 数字 | True  a| 
JSON 数字，表示在试用订阅结束之前客户通知将开始的天数。该值以天表示，值为 -1 表示不适用。



| onboardStatus | string | True  a| 
JSON 字符串，指示客户入职的当前状态。有效值为 "not started"、"in progress"、"success"、"failed"。定义的值为：

* "未启动"
* "正在进行中"
* "成功"
* "失败"




| costPerAppUnit | 数字 | True  a| 
表示每个应用单位成本的 JSON 数字。对于试用版，值为 0。单位：美元



| costPerNamespaceUnit | 数字 | True  a| 
表示每个命名空间单位成本的 JSON 数字。对于试用版，值为 0。单位：美元



| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。
