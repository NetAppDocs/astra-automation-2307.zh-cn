= 修改存储桶
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-put]#PUT# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/buckets/{bucket_id}`#

将整个存储资源替换为提供的 JSON 请求正文，同时保留不可由用户修改的键值。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| bucket_id | string | path | True  a| 
要列出的存储区集合的 ID

|===


== 请求正文

将整个存储资源替换为提供的 JSON 请求正文，同时保留不可由用户修改的键值。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-bucket"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"
* "1.2"




| id | string | False  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | False  a| 
包含用户分配的存储桶名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：256




| 凭证 ID | string | False  a| 
包含用于访问存储区的凭据标识符的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| 状态 | string | False  a| 
包含指示存储桶操作状态的值的 JSON 字符串。定义的值为：

* "pending" - 已计划发现存储桶
* "available" - 存储桶可供使用
* "failed" - 存储桶处于永久失败状态
* "removed" - 存储桶当前不可访问
* "unknown" - 无法确定存储桶状态




| stateDetails | 数组[link:#type_astra_stateDetail["type_astra_stateDetail"]] | False  a| 
JSON 对象的 JSON 数组，列出有关"state"转换为"removed"或"failed"的其他详细信息。如果没有可用的详细信息，此数组将为空。

* 最小项目数：0




| 提供商 | string | False  a| 
JSON 字符串，表示可用于访问存储桶的存储桶协议。定义的值为：

* "ontap-s3"
* "storagegrid-s3"
* “generic-s3”
* "gcp"
* "azure"
* "aws"




| bucketParameters | link:#bucketParameters["bucketParameters"] | False  a| 
提供访问存储桶所需的存储桶提供程序特定信息的 JSON 对象。此存储桶中的特定值因提供程序而异。



| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | False  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。


|===
.请求示例
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-bucket",
  "version": "1.2",
  "name": "New Bucket Name"
}
----
====
