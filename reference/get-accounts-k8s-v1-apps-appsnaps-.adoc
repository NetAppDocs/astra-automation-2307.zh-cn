= 检索应用程序快照
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/k8s/v1/apps/{app_id}/appSnaps/{appSnap_id}`#

返回 JSON 响应正文中存储的 Application Snapshot API 资源。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| app_id | string | path | True  a| 
包含应用程序资源的 ID



| appSnap_id | string | path | True  a| 
要列出的 appSnap 集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns the stored Application Snapshot API resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-appSnap"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"
* "1.2"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | True  a| 
包含快照的用户指定名称的 JSON 字符串。如果未指定，或者当快照由计划启动时，系统会分配快照的名称。定义的值为：

* 最小长度：1
* 最大长度：63
* 符合 DNS 1123 标签架构




| scheduleID | string | False  a| 
包含可选计划 ID 的 JSON 字符串，指示快照是由计划触发的，并指定哪个计划触发了快照。仅适用于由计划启动的快照。定义的值为：

* 符合 Astra 标识符架构




| snapshotAppAsset | string | False  a| 
包含与快照对应的应用程序资产 ID 的 JSON 字符串。当应用程序快照的状态为"completed"时，将返回此字段。定义的值为：

* 符合 Astra 标识符架构




| 状态 | string | True  a| 
包含指示快照操作状态的值的 JSON 字符串。定义的值为：

* "pending" - 应用程序快照计划进行发现
* "discovering" - 正在发现应用程序快照
* "running" - 应用快照运行正常
* "completed" - 应用程序快照已完成
* "failed" - 应用程序快照处于永久失败状态
* "removed" - 当前无法访问应用程序快照
* "unknown" - 无法确定应用程序快照状态




| stateUnready | 数组[string] | True  a| 
JSON 字符串的 JSON 数组，每个数组指示应用程序快照无法转换为"running"或转换为"failed"的原因。定义的值为：

* 最小长度：1
* 最大长度：127




| hookState | string | False  a| 
JSON 字符串包含一个值，该值指示作为快照的一部分运行的任何一个或多个执行挂钩的状态。定义的值为：

* "failed" - 一个或多个执行挂钩出现故障
* "success" - 没有或多个适用的挂钩，所有挂钩均成功




| hookStateDetails | 数组[link:#type_astra_stateDetail["type_astra_stateDetail"]] | False  a| 
JSON 对象的 JSON 数组，其中列出了有关"hookState"转换为"失败"的其他详细信息。如果没有可用的详细信息，此数组将为空。

* 最小项目数：0




| 元数据 | link:#type_astra_metadata["type_astra_metadata"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。
