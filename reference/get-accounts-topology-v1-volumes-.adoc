= 检索卷
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/volumes/{volume_id}`#

返回 JSON 响应正文中存储的 Volume API 资源。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| volume_id | string | path | True  a| 
要列出的卷集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns the stored Volume API resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-volume"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"
* "1.2"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | True  a| 
包含卷名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：255




| 状态 | string | False  a| 
JSON 字符串，包含指示卷操作状态的值。



| 大小 | string | False  a| 
包含卷大小的 JSON 字符串。



| 已用 | 数字 | False  a| 
包含已配置卷使用的配额字节数的 JSON 数字。



| usedPercentage | 数字 | False  a| 
包含已用卷百分比的 JSON 数字。



| 总计 | 数字 | False  a| 
包含已配置卷的总字节数的 JSON 编号。



| creationToken | string | False  a| 
包含卷的创建令牌的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：255




| snapshotPossible | string | False  a| 
JSON 字符串，指示驱动程序是否可以执行卷的快照。定义的值为：

* "true" - 驱动程序可以获取卷的快照
* "false" - 驱动程序无法获取卷的快照




| storageClass | string | False  a| 
包含卷的存储类的 JSON 字符串。

* 定义的值为：
* 最小长度：1
* 最大长度：255




| pvcName | string | False  a| 
包含卷的 PersistentVolumeClaim (PVC) 名称的 JSON 字符串。

* 定义的值为：
* 最小长度：1
* 最大长度：255




| internalName | string | False  a| 
包含目标存储群集上卷的内部名称的 JSON 字符串（如果可用）。此名称来自 Kubernetes CSI 驱动程序。

* 定义的值为：
* 最小长度：1
* 最大长度：255




| appsUsing | 数组[string] | True  a| 
JSON 字符串的 JSON 数组，每个数组都包含正在使用该卷的应用程序的 ID。定义的值为：

* 符合 Astra 标识符架构




| storageBackendID | string | False  a| 
JSON 字符串，包含提供卷的存储后端的 ID。定义的值为：

* 符合 Astra 标识符架构




| 编排器 | string | False  a| 
包含此卷的编排器的 JSON 字符串。这将为 Astra Data Store 卷定义。定义的值为：

* "trident" - Trident 已配置此卷
* "vasa" - NetApp VASA Provider 已配置此卷
* "other" - 已手动配置卷或 orchestrator 未知




| serviceLevel | string | False  a| 
包含在存储后端级别为此卷定义的服务质量 (QoS) 策略的 JSON 字符串。这将为 Astra Data Store 卷定义。



| exportPolicy | string | False  a| 
包含与此卷关联的导出策略的 JSON 字符串。这将为 Astra Data Store 卷定义。



| nodeId | string | False  a| 
包含为卷提供服务的节点 ID 的 JSON 字符串。这将为 Astra Data Store 卷定义。定义的值为：

* 符合 Astra 标识符架构




| healthState | string | False  a| 
JSON 字符串，包含指示卷健康状态的值。这将为 Astra Data Store 卷定义。定义的值为：

* "不确定" - 无法确定资源的健康状况（例如，由于缺乏连接）
* "normal" - 资源没有已知问题，可以根据接口规范提供服务
* "warning" - 资源存在已知问题，但不会阻止其根据接口规范提供服务
* "critical" - 资源存在已知问题，无法根据接口规范提供服务




| healthStateDetails | 数组[link:#type_astra_stateDetail["type_astra_stateDetail"]] | True  a| 
JSON 对象的 JSON 数组，列出有关当前运行状况的其他详细信息。如果没有可用的详细信息，此数组将为空。

* 最小项目数：0




| 元数据 | link:#type_astra_metadata["type_astra_metadata"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构
