= 列出组的用户
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/groups/{group_id}/users`#

返回集合中所有匹配资源的 JSON 数组。如果在请求中指定了单个字段，则项目 JSON 数组将包含按指定顺序为每个匹配资源请求的字段。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| group_id | string | path | True  a| 
包含的组资源的 ID



| 包括 | string | 查询 | False  a| 
指示列出集合时应返回哪些字段。

URL 示例："include=id"、"include=id,name"

OpenAPI 示例："id"

* 示例：id、email




| limit | string | 查询 | False  a| 
指示列出集合时要返回的最大资源数。

URL 示例："limit=2"

OpenAPI 示例："2"



| 筛选器 | string | 查询 | False  a| 
指示在列出集合时应匹配哪些字段以返回资源。

支持的运算符有：

* eq - 相等
* lt/gt - 小于/大于
* lte/gte - 小于或等于/大于或等于


URL 示例："filter=field%20eq%20%27value%27"

OpenAPI 示例："field eq 'value'"



| 订单 | string | 查询 | False  a| 
指示列出集合时资源的排序顺序。

URL 示例："orderBy=name"，"orderBy=name%20desc"

OpenAPI 示例："name desc"



| 跳过 | string | 查询 | False  a| 
指示列出集合时要跳过的资源数。

URL 示例："skip=1"

OpenAPI 示例："1"



| count | string | 查询 | False  a| 
指示是否应在元数据对象中返回资源总数。

URL 示例："count=true"

OpenAPI 示例："true"



| 继续 |  | 查询 | False  a| 
指示要从中恢复列表的令牌。

URL 示例："continue=+++<continue token="" from="" collection="" metadata="">+++"+++</continue>+++

OpenAPI 示例："+++<continue token="" from="" collection="" metadata="">+++</continue>+++</continue>

|===

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-user"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"
* "1.2"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| 状态 | string | True  a| 
包含指示用户操作状态的值的 JSON 字符串。定义的值为：

* "pending" - 正在创建用户帐户。用户只能读取和修改用户资源（默认值）。
* 此状态对于 "authProvider" 为 "local" 的用户无效
* "active" - 用户帐户是可操作的（如果帐户设置"isEnabled"为"true"，则用户可以登录）。
* 这是 "authProvider" 为 "local" 的用户的默认值
* "suspended" - 用户帐户已暂停（无论帐户 "isEnabled" 设置如何，用户都无法登录）。




| isEnabled | string | True  a| 
指示是否允许用户登录的 JSON 字符串。定义的值为：

* "true" - 允许用户登录；创建时为默认值。
* "false" - 不允许此用户登录。




| authID | string | False  a| 
包含与外部标识提供程序关联的标识符的 JSON 字符串。此字符串的内容取决于指定的 authProvider。对于 "authProvider" 为 "local" 的用户，"authID" 设置为 "email" 的值。对于 "authProvider" 为 "ldap" 的用户，"authID" 设置为其来自 LDAP 服务器的可分辨名称 (DN)。



| authProvider | string | False  a| 
表示 idAuth 标识符上下文的 JSON 字符串。定义的值为：

* "cloud-central" - 用户身份验证来自 NetApp Cloud Central 服务
* "local" - 用户身份验证来自本地用户/密码存储
* "ldap" - 用户身份验证来自外部 LDAP 服务器。在 Astra Control Center 中，所有用户都必须具有 "local" 或 "ldap" 值。如果未指定值，则选择默认值 "local"。在 Astra Control Service 中，不支持将 "authProvider" 设为 "local" 或 "ldap"。如果未指定值，则默认为 "cloud-central"。




| firstName | string | True  a| 
包含用户指定的长格式名字的 JSON 字符串。定义的值为：

* 最小长度：0
* 最大长度：63
* 此字符串经过内部验证，可防止 XSS、Unicode、目录遍历和 SQL 注入攻击




| lastName | string | True  a| 
包含用户指定的长格式姓氏的 JSON 字符串。定义的值为：

* 最小长度：0
* 最大长度：63
* 此字符串经过内部验证，可防止 XSS、Unicode、目录遍历和 SQL 注入攻击




| companyName | string | False  a| 
包含与用户关联的公司名称的 JSON 字符串。如果此密钥不存在，则表示未提供公司名称。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，可防止 XSS、Unicode、目录遍历和 SQL 注入攻击




| 电子邮件 | string | True  a| 
包含用户指定的电子邮件地址的 JSON 字符串。



| 电话 | string | False  a| 
包含用户电话号码的 JSON 字符串。如果此密钥不存在，则表示未提供电话号码。



| postalAddress | link:#postalAddress["postalAddress"] | False  a| 
JSON 对象包含指定地址的字段集合。如果此 key 不存在，则表示未提供地址。如果此 key 存在，则所有六个包含的字段都将存在。如果用户未提供任何内容，则字符串将为空。



| sendWelcomeEmail | string | True  a| 
表示是否在验证后发送欢迎电子邮件的 JSON 字符串。如果在创建时未指定，则 ACS 的值将设置为 "true"，Astra Control Center 的值将设置为 "false"。定义的值为：

* "true" - 将发送一封欢迎电子邮件；创建时为默认值
* "false" - 不会发送欢迎电子邮件。对于 "authProvider" 为 "local" 或 "ldap" 的用户，此值将被忽略并设置为 "false"。




| enableTimestamp | string | False  a| 
包含 ISO-8601 格式时间戳的 JSON 字符串，指示上次启用用户的时间。定义的值为：

* 符合 ISO-8601 日期时间方案




| lastActTimestamp | string | False  a| 
包含 ISO-8601 格式时间戳的 JSON 字符串，指示上次代表此用户调用身份 API 的时间。（注：这是昂贵的。）定义的值为：

* 符合 ISO-8601 日期时间方案




| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。
