= 创建新包
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-post]#POST# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/packages`#

指示要创建的 Package API 资源的所需值。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}


|===


== 请求正文

指示要创建的 Package API 资源的所需值。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-package"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| packageName | string | True  a| 
包含包名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：31 示例值为：
* "acc"
* "acs"
* “trident”




| packageVersion | string | True  a| 
包含包的 SemVer 的 JSON 字符串。定义的值为：

* 符合 SemVer 架构




| packageType | string | True  a| 
包含程序包类型的 JSON 字符串，指示这是完整的安装程序包还是修补程序。定义的值为：

* "安装"
* “补丁”




| bundleName | 数组[string] | False  a| 
包含软件包所在捆绑包名称的 JSON 字符串的 JSON 数组。



| severityLevel | string | True  a| 
描述此修补程序或安装的严重级别的 JSON 字符串。定义的值为：

* "recommended" - 建议安装 - 默认值
* "critical" - 强烈建议安装以解决关键错误修复和/或安全修补程序




| 镜像 | 数组[link:#images["镜像"]] | False  a| 
JSON 对象的 JSON 数组，每个对象都包含作为包一部分的 Docker 镜像。



| 制品 | 数组[link:#artifacts["制品"]] | False  a| 
描述与此包关联的其他工件（二进制文件和其他）的 JSON 对象的 JSON 数组。



| 文件 | 数组[link:#files["文件"]] | False  a| 
JSON 对象的 JSON 数组，每个表示与此包关联的非图像制品。



| upgradableVersions | link:#upgradableVersions["upgradableVersions"] | False  a| 
JSON 对象包含可以升级到包版本的最低和最高版本。



| 依赖项 | 数组[link:#dependencies["依赖项"]] | False  a| 
描述安装此包所需的给定依赖项最低版本的 JSON 对象的 JSON 数组



| 元数据 | link:#type_astra_metadata["type_astra_metadata"] | False  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。
