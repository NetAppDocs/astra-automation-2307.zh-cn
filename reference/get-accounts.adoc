= 列出所有帐户
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts`#

返回集合中所有匹配资源的 JSON 数组。如果在请求中指定了单个字段，则项目 JSON 数组将包含按指定顺序为每个匹配资源请求的字段。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| 包括 | string | 查询 | False  a| 
指示列出集合时应返回哪些字段。

URL 示例："include=id"、"include=id,name"

OpenAPI 示例："id"



| limit | string | 查询 | False  a| 
指示列出集合时要返回的最大资源数。

URL 示例："limit=2"

OpenAPI 示例："2"



| 筛选器 | string | 查询 | False  a| 
指示在列出集合时应匹配哪些字段以返回资源。

支持的运算符有：

* eq - 相等
* lt/gt - 小于/大于
* lte/gte - 小于或等于/大于或等于


URL 示例："filter=field%20eq%20%27value%27"

OpenAPI 示例："field eq 'value'"



| 订单 | string | 查询 | False  a| 
指示列出集合时资源的排序顺序。

URL 示例："orderBy=name"，"orderBy=name%20desc"

OpenAPI 示例："name desc"



| 跳过 | string | 查询 | False  a| 
指示列出集合时要跳过的资源数。

URL 示例："skip=1"

OpenAPI 示例："1"



| count | string | 查询 | False  a| 
指示是否应在元数据对象中返回资源总数。

URL 示例："count=true"

OpenAPI 示例："true"



| 继续 |  | 查询 | False  a| 
指示要从中恢复列表的令牌。

URL 示例："continue=+++<continue token="" from="" collection="" metadata="">+++"+++</continue>+++

OpenAPI 示例："+++<continue token="" from="" collection="" metadata="">+++</continue>+++</continue>

|===

== 定义

.查看定义
[%collapsible]
====
[role="api-collapsible-fifth-title"]
postalAddress

包含指定地址的 JSON 字符串集合的 JSON 对象。如果此 key 不存在，则表示未提供地址。如果此键存在，则所有六个包含的 JSON 字符串都将存在。如果不提供任何内容，字符串将为空。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| addressCountry | string | True  a| 
包含与邮政地址关联的国家/地区名称或国家/地区代码的 JSON 字符串，如 ISO 3166 Alpha-2 中所指定。定义的值为：

* 最小长度：2
* 最大长度：2




| addressLocality | string | True  a| 
包含与邮政地址关联的地点（例如城市）的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| addressRegion | string | True  a| 
包含与邮政地址关联的区域（例如州）的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| postalCode | string | True  a| 
包含与邮政地址关联的邮政编码的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：31




| streetAddress1 | string | True  a| 
包含与邮政地址关联的街道地址的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| streetAddress2 | string | False  a| 
包含可选辅助街道地址信息、邮政信箱等的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63


|===
[role="api-collapsible-fifth-title"]
accountContact

JSON 对象包含用户联系信息，用于在帐户激活时创建所有者用户。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| firstName | string | True  a| 
包含用户指定的长格式名字的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，以防止 XSS、Unicode、目录遍历和 SQL 注入攻击。




| lastName | string | True  a| 
包含用户指定的长格式姓氏的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，以防止 XSS、Unicode、目录遍历和 SQL 注入攻击。




| companyName | string | False  a| 
包含与用户关联的公司名称的 JSON 字符串。如果此密钥不存在，则表示未提供公司名称。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，以防止 XSS、Unicode、目录遍历和 SQL 注入攻击。




| 电子邮件 | string | True  a| 
包含用户指定的电子邮件地址的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| 电话 | string | False  a| 
包含用户电话号码的 JSON 字符串。如果此密钥不存在，则表示未提供电话号码。定义的值为：

* 最小长度：1
* 最大长度：31




| postalAddress | link:#postalAddress["postalAddress"] | True  a| 
包含指定地址的 JSON 字符串集合的 JSON 对象。如果此 key 不存在，则表示未提供地址。如果此键存在，则所有六个包含的 JSON 字符串都将存在。如果不提供任何内容，字符串将为空。

* 符合 ISO-8601 日期时间方案
