= 检索挂钩源
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/hookSources/{hookSource_id}`#

返回 JSON 响应正文中存储的 Hook Source API 资源。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| hookSource_id | string | path | True  a| 
要列出的 hookSource 集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns the stored Hook Source API resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-hookSource"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | True  a| 
包含用户指定的资源名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63 在所有挂钩源资源中，名称必须是唯一的。




| 专用 | string | True  a| 
JSON 字符串，指示源字段是在 API 中公开还是有意留空。定义的值为：

* "true" - 源值是私有的，不会通过 API 公开。
* "false" - 源值不是私有的，将通过 API 公开。




| 预加载 | string | True  a| 
用于指示源是否已预加载的 JSON 字符串。定义的值为：

* "true" - 源已预加载。
* "false" - 未预加载源。预加载的挂钩源资源为只读。




| sourceType | string | True  a| 
定义钩子源类型的 JSON 字符串。定义的值为：

* "script" - Shell 脚本。明文源不得包含二进制数据或回车字符，否则返回错误。




| 源 | string | False  a| 
包含要以 base64 格式运行的脚本的 JSON 字符串。定义的值为：

* 最小长度：0
* 最大长度：131072
* 符合 base64 编码数据架构，使用 base64 编码时，源值的最大大小为 128 KiB。




| sourceMD5Checksum | string | True  a| 
包含为"source"键值计算的 MD5 校验和的 JSON 字符串。定义的值为：

* 定义的正则表达式为："^([0-9a-fA-F]){32}$"




| description | string | False  a| 
包含用户指定的钩子源描述的 JSON 字符串。定义的值为：

* 最小长度：0
* 最大长度：511




| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。
