= 列出受管集群的命名空间
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/managedClusters/{managedCluster_id}/namespaces`#

返回集合中所有匹配资源的 JSON 数组。如果在请求中指定了单个字段，则项目 JSON 数组将包含按指定顺序为每个匹配资源请求的字段。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| managedCluster_id | string | path | True  a| 
包含 managedCluster 资源的 ID



| 包括 | string | 查询 | False  a| 
指示列出集合时应返回哪些字段。

URL 示例："include=id"、"include=id,name"

OpenAPI 示例："id"

* 示例：id、name、kubernetesLabels




| limit | string | 查询 | False  a| 
指示列出集合时要返回的最大资源数。

URL 示例："limit=2"

OpenAPI 示例："2"



| 筛选器 | string | 查询 | False  a| 
指示在列出集合时应匹配哪些字段以返回资源。

支持的运算符有：

* eq - 相等
* lt/gt - 小于/大于
* lte/gte - 小于或等于/大于或等于


URL 示例："filter=field%20eq%20%27value%27"

OpenAPI 示例："field eq 'value'"

|===

包含命名空间所在的群集的全局唯一标识符的 JSON 字符串。定义的值为：

| systemType | string | False  a| 
表示命名空间是系统命名空间类型的 JSON 字符串。此字段标识命名空间是什么类型的系统命名空间（如果有的话）（仅版本 1.1）。定义的值为：

* "kubernetes" - 一个通用的 Kubernetes (K8s) 系统命名空间，例如 "kube-system"
* "netapp" - NetApp 系统命名空间，例如 "trident" 或 "netapp-monitoring"
* "openshift" - 一个 OpenShift 系统命名空间，例如 "openshift-monitoring"
* "rke" - Rancher 命名空间，例如 "cattle-logging"
* "other" - 不一定适合其他地方的通用系统命名空间，例如 "ingress-nginx"、"tekton-system"
