= 列出所有任务
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/tasks`#

返回集合中所有匹配资源的 JSON 数组。如果在请求中指定了单个字段，则项目 JSON 数组将包含按指定顺序为每个匹配资源请求的字段。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| 包括 | string | 查询 | False  a| 
指示列出集合时应返回哪些字段。

URL 示例："include=id"、"include=id,name"

OpenAPI 示例："id"



| limit | string | 查询 | False  a| 
指示列出集合时要返回的最大资源数。

URL 示例："limit=2"

OpenAPI 示例："2"



| 筛选器 | string | 查询 | False  a| 
指示在列出集合时应匹配哪些字段以返回资源。

支持的运算符有：

* eq - 相等
* lt/gt - 小于/大于
* lte/gte - 小于或等于/大于或等于


URL 示例："filter=field%20eq%20%27value%27"

OpenAPI 示例："field eq 'value'"

|===

| type | string | True  a| 
资源的媒体类型。定义的值为：

资源的版本。定义的值为：

* "1.0"

| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构

包含任务的点符号 (.) 层次结构名称的 JSON 字符串。定义的值为：

* 最小长度：3
* 最大长度：127
* 定义的正则表达式为："^(([a-z])_(\.))_([a-z])+$"




| 摘要 | string | True  a| 
包含任务的短格式摘要的 JSON 字符串。定义的值为：

* 最小长度：3
* 最大长度：63




| description | string | True  a| 
包含任务说明的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：511




| service | string | False  a| 
包含管理任务的服务的名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：31




| parentTaskID | string | False  a| 
包含任务的父任务的全局唯一标识符的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| 用户 ID | string | False  a| 
包含与任务关联的用户 ID 的 JSON 字符串。仅包含与用户启动的操作相关联的任务。定义的值为：

* 符合 Astra 标识符架构




| resourceID | string | True  a| 
包含与任务关联的资源的全局唯一标识符的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| 资源 URI | string | True  a| 
包含与任务关联的 URI 的 JSON 字符串。定义的值为：

* 最小长度：3
* 最大长度：4095




| resourceCollectionURI | 数组[string] | True  a| 
JSON 字符串的 JSON 数组，每个数组指示与任务关联的资源的完全扩展的访问 URL。定义的值为：

* 最小长度：3
* 最大长度：4095




| 状态 | string | True  a| 
包含指示任务操作状态的值的 JSON 字符串。定义的值为：

* "notStarted" - 任务尚未开始执行。
* "running" - 任务正在运行。
* "completed" - 任务已完成。
* "pausing" - 请求暂停任务。
* "paused" - 任务已暂停。
* "cancelling" - 任务被请求取消。
* "cancelled" - 任务已取消。
* "failed" - 任务失败。
* 最小长度：1
* 最大长度：31




| stateTransitions | 数组[link:#type_astra_stateTransition["type_astra_stateTransition"]] | True  a| 
JSON 对象的 JSON 数组，每个表示从给定定义状态的允许状态转换集。每个 JSON 对象都包含"from"和"to"字段，其中"from"值是状态的名称，"to"值是允许的目标状态数组。

* 最小长度：1
* 最大长度：31 例如："stateTransitions" : [ { "from": "running", "to": ["paused", "cancelled"] }, { "from": "paused", "to": ["running", "cancelled"] }, ]




| stateDetails | 数组[link:#type_astra_stateDetail["type_astra_stateDetail"]] | True  a| 
列出有关当前"状态"的其他详细信息的 JSON 对象的 JSON 数组。此字段指示何时存在阻止实际状态与所需状态匹配的条件或错误。如果没有可用的详细信息，此数组将为空。

* 最小项目数：0




| orderHint | 数字 | False  a| 
表示子任务顺序的 JSON 编号。客户端可以使用此字段将子任务从最小到最大排序。



| percentDone | 数字 | False  a| 
包含正在运行的任务的完成百分比的 JSON 编号。任务完成后，设置为"100"。



| startTime | string | False  a| 
包含指示任务何时启动的时间戳的 JSON 字符串。定义的值为：

* 符合 ISO-8601 日期时间方案




| endTime | string | False  a| 
包含指示任务何时结束的时间戳的 JSON 字符串。定义的值为：

* 符合 ISO-8601 日期时间方案




| cancelTime | string | False  a| 
包含指示任务何时被取消的时间戳的 JSON 字符串。定义的值为：

* 符合 ISO-8601 日期时间方案
