= 检索受管集群的命名空间
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/managedClusters/{managedCluster_id}/namespaces/{namespace_id}`#

返回 JSON 响应正文中存储的 Namespace API 资源。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| managedCluster_id | string | path | True  a| 
包含 managedCluster 资源的 ID



| 命名空间 ID | string | path | True  a| 
要列出的命名空间集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns the stored Namespace API resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-namespace"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| 链接 | 数组[link:#type_astra_link["type_astra_link"]] | True  a| 
描述针对资源的支持操作的链接。



| name | string | True  a| 
包含指示命名空间名称的值的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：255




| namespaceState | string | True  a| 
包含指示命名空间状态的值的 JSON 字符串。定义的值为：

* "discovered" - 已在集群中找到命名空间
* "removed" - 命名空间已从集群中删除，或集群未被管理，但可能存在引用命名空间的资源




| namespaceStateDetails | 数组[link:#type_astra_stateDetail["type_astra_stateDetail"]] | True  a| 
JSON 对象的 JSON 数组，列出有关"state"转换为"state a"、"state b"或"state c"的其他详细信息。如果没有可用的详细信息，此数组将为空。定义的值为：

* 最小项目数：0




| kubernetesLabels | 数组[link:#type_astra_label["类型 Astra 标签"]] | False  a| 
JSON 对象的 JSON 数组，每个都包含与命名空间关联的 Kubernetes 标签。定义的值为：

* 符合 Kubernetes 标签数组架构




| clusterID | string | True  a| 
包含命名空间所在的群集的全局唯一标识符的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| systemType | string | False  a| 
表示命名空间是系统命名空间类型的 JSON 字符串。此字段标识命名空间是什么类型的系统命名空间（如果有的话）（仅版本 1.1）。定义的值为：

* "kubernetes" - 一个通用的 Kubernetes (K8s) 系统命名空间，例如 "kube-system"
* "netapp" - NetApp 系统命名空间，例如 "trident" 或 "netapp-monitoring"
* "openshift" - 一个 OpenShift 系统命名空间，例如 "openshift-monitoring"
* "rke" - Rancher 命名空间，例如 "cattle-logging"
* "other" - 不一定适合其他地方的通用系统命名空间，例如 "ingress-nginx"、"tekton-system"




| 元数据 | link:#type_astra_metadata["type_astra_metadata"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构
