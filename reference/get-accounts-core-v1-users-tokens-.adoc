= 检索令牌
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/users/{user_id}/tokens/{token_id}`#

返回 JSON 响应正文中存储的 Token API 资源。

* 不会返回 "token" 键/值。




== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| 用户 ID | string | path | True  a| 
包含用户资源的 ID



| token_id | string | path | True  a| 
要列出的令牌集合的 ID

|===


== 响应

[source]
----
Status: 200
----
返回 JSON 响应正文中存储的 Token API 资源。

* 不会返回 "token" 键/值。


[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-token"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | True  a| 
包含令牌名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，以防止 XSS、Unicode、目录遍历和 SQL 注入攻击。




| 用户 ID | string | True  a| 
包含与令牌关联的用户的 UUID 的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。


|===

====

{
  "type": "application/astra-token",
  "version": "1.0",
  "id": "822fab12-c224-5d1f-8118-5fb5d2def8b9",
  "name": "Snapshot Script",
  "userID": "09f8933c-ad74-4f4e-8ef5-1ffaa0fb8e9b",
  "metadata": {
    "labels": [],
    "creationTimestamp": "2022-10-06T20:58:16.305662Z",
    "modificationTimestamp": "2022-10-06T20:58:16.305662Z",
    "createdBy": "8f84cf09-8036-51e4-b579-bd30cb07b269"
  }
}
----
====
