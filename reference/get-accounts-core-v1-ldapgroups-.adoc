= 检索 LDAP 组
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/ldapGroups/{ldapGroup_id}`#

返回表示存储的 LDAP 资源的 JSON 对象。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| ldapGroup_id | string | path | True  a| 
要列出的 ldapGroup 集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns a JSON object representing the stored LDAP resource.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-ldapGroup"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| cn | string | False  a| 
表示 LDAP 组名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| dn | string | False  a| 
表示 LDAP 组可分辨名称的 JSON 字符串。定义的值为：

* 符合 RFC 4514




| 元数据 | link:#type_astra_metadata["type_astra_metadata"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构


|===
