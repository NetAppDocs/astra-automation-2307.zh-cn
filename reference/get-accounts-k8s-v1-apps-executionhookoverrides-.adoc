= 检索执行钩子覆盖资源
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/k8s/v1/apps/{app_id}/executionHookOverrides/{executionHookOverride_id}`#

返回 JSON 响应正文中存储的 Execution Hook Override API 资源。


CAUTION: 此操作已弃用。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| app_id | string | path | True  a| 
包含应用程序资源的 ID



| executionHookOverride_id | string | path | True  a| 
要列出的 executionHookOverride 集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns the stored Execution Hook Override API resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-executionHookOverride"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| executionHookID | string | True  a| 
包含执行挂钩 ID 的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构 仅允许 hookType 值为 "netapp" 的执行挂钩资源的 executionHookID。




| 启用 | string | True  a| 
指示是否为指定应用程序启用了执行挂钩的 JSON 字符串。定义的值为：

* "true" - 执行挂钩已启用，将按指定顺序运行
* "false" - 执行挂钩已禁用，无法运行




| appID | string | True  a| 
包含重写链接的应用程序的关联 ID 的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。


|===
