= 修改组中用户的令牌
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-put]#PUT# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/groups/{group_id}/users/{user_id}/tokens/{token_id}`#

将整个存储资源替换为提供的 JSON 请求正文，同时保留不可由用户修改的键值。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| group_id | string | path | True  a| 
包含的组资源的 ID



| 用户 ID | string | path | True  a| 
包含用户资源的 ID



| token_id | string | path | True  a| 
要列出的令牌集合的 ID

|===


== 请求正文

将整个存储资源替换为提供的 JSON 请求正文，同时保留不可由用户修改的键值。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-token"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| id | string | False  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | False  a| 
包含令牌名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，以防止 XSS、Unicode、目录遍历和 SQL 注入攻击。




| 用户 ID | string | False  a| 
包含与令牌关联的用户的 UUID 的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | False  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。


|===
.请求示例
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-token",
  "version": "1.0",
  "name": "New Token Name"
}
----
====
