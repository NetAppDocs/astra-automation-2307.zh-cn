= 为用户创建新组
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-post]#POST# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/users/{user_id}/groups`#

指示要创建的 Group API 资源的所需值。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| 用户 ID | string | path | True  a| 
包含用户资源的 ID

|===


== 请求正文

指示要创建的 Group API 资源的所需值。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-group"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"




| name | string | False  a| 
组资源的名称。如果在创建时未指定，则该值将设置为

* 如果在 authID 中找到第一个公用名 (CN) 相对可分辨名称 (RDN)，
* authID，否则。如果在更新时未指定，则会保留该值，不进行任何修改。定义的值为：
* 最小长度：1
* 最大长度：2048




| authProvider | string | True  a| 
组资源的身份验证提供程序。定义的值为：

* “ldap”




| authID | string | True  a| 
LDAP 组的可分辨名称 (DN)。定义的值为：

* 最小长度：1
* 最大长度：2048




| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | False  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。


|===
.请求示例
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-group",
  "version": "1.1",
  "name": "engineering-group",
  "authProvider": "ldap",
  "authID": "CN=Engineering,CN=Groups,DC=example,DC=com"
}
----
====
