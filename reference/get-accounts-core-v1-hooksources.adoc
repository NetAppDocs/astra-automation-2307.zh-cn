= 列出所有 hook 源
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/hookSources`#

返回集合中所有匹配资源的 JSON 数组。如果在请求中指定了单个字段，则项目 JSON 数组将包含按指定顺序为每个匹配资源请求的字段。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| 包括 | string | 查询 | False  a| 
指示列出集合时应返回哪些字段。

URL 示例："include=id"、"include=id,name"

OpenAPI 示例："id"



| limit | string | 查询 | False  a| 
指示列出集合时要返回的最大资源数。

URL 示例："limit=2"

OpenAPI 示例："2"



| 筛选器 | string | 查询 | False  a| 
指示在列出集合时应匹配哪些字段以返回资源。

支持的运算符有：

* eq - 相等
* lt/gt - 小于/大于
* lte/gte - 小于或等于/大于或等于


URL 示例："filter=field%20eq%20%27value%27"

OpenAPI 示例："field eq 'value'"

|===

包含用户指定的资源名称的 JSON 字符串。定义的值为：

| 专用 | string | True  a| 
JSON 字符串，指示源字段是在 API 中公开还是有意留空。定义的值为：

* "true" - 源值是私有的，不会通过 API 公开。
* "false" - 源值不是私有的，将通过 API 公开。




| 预加载 | string | True  a| 
用于指示源是否已预加载的 JSON 字符串。定义的值为：

* "true" - 源已预加载。
* "false" - 未预加载源。预加载的挂钩源资源为只读。




| sourceType | string | True  a| 
定义钩子源类型的 JSON 字符串。定义的值为：

* "script" - Shell 脚本。明文源不得包含二进制数据或回车字符，否则返回错误。




| 源 | string | False  a| 
包含要以 base64 格式运行的脚本的 JSON 字符串。定义的值为：

* 最小长度：0
* 最大长度：131072
* 符合 base64 编码数据架构，使用 base64 编码时，源值的最大大小为 128 KiB。




| sourceMD5Checksum | string | True  a| 
包含为"source"键值计算的 MD5 校验和的 JSON 字符串。定义的值为：

* 定义的正则表达式为："^([0-9a-fA-F]){32}$"




| description | string | False  a| 
包含用户指定的钩子源描述的 JSON 字符串。定义的值为：

* 最小长度：0
* 最大长度：511
