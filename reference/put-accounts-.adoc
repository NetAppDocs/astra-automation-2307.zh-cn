= 修改账户
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-put]#PUT# [.api-doc-code-block]#`/accounts/{account_id}`#

将整个存储资源替换为提供的 JSON 请求正文，同时保留不可由用户修改的键值。

*副作用*

* 将 "isEnabled" 从 "false" 更改为 "true" 会将 "enabledTimestamp" 设置为当前时间。




== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
要列出的帐户集合的 ID

* 示例：{{.Account}}


|===


== 请求正文

将整个存储资源替换为提供的 JSON 请求正文，同时保留不可由用户修改的键值。

*副作用*

* 将 "isEnabled" 从 "false" 更改为 "true" 会将 "enabledTimestamp" 设置为当前时间。


[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-account"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| id | string | False  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | False  a| 
包含帐户名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，以防止 XSS、Unicode、目录遍历和 SQL 注入攻击。




| 状态 | string | False  a| 
包含指示帐户操作状态的值的 JSON 字符串。定义的值为：

* "pending" - 只能对帐户下的用户和角色资源进行更改（默认值）
* "active" - 可以对帐户下的任何资源进行更改
* "deletePending" - 用户无法登录，数据保护操作暂停，费用不再累计




| isEnabled | string | False  a| 
表示已启用帐户的 JSON 字符串。定义的值为：

* "false" - 不允许用户使用此帐户
* "true" - 允许用户使用此帐户




| enabledTimestamp | string | False  a| 
JSON 字符串，包含指示上次启用帐户的时间的时间戳。如果帐户从未启用，则此字段不存在。定义的值为：

* 符合 ISO-8601 日期时间方案




| accountContact | link:#accountContact["accountContact"] | False  a| 
JSON 对象包含用户联系信息，用于在帐户激活时创建所有者用户。



| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | False  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。

== 定义

[role="api-collapsible-fifth-title"]
postalAddress

包含指定地址的 JSON 字符串集合的 JSON 对象。如果此 key 不存在，则表示未提供地址。如果此键存在，则所有六个包含的 JSON 字符串都将存在。如果不提供任何内容，字符串将为空。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| addressCountry | string | False  a| 
包含与邮政地址关联的国家/地区名称或国家/地区代码的 JSON 字符串，如 ISO 3166 Alpha-2 中所指定。定义的值为：

* 最小长度：2
* 最大长度：2




| addressLocality | string | False  a| 
包含与邮政地址关联的地点（例如城市）的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| addressRegion | string | False  a| 
包含与邮政地址关联的区域（例如州）的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| postalCode | string | False  a| 
包含与邮政地址关联的邮政编码的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：31




| streetAddress1 | string | False  a| 
包含与邮政地址关联的街道地址的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63




| streetAddress2 | string | False  a| 
包含可选辅助街道地址信息、邮政信箱等的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：63
