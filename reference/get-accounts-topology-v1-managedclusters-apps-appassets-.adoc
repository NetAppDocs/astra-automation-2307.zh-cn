= 检索受管集群的应用程序资产
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/topology/v1/managedClusters/{managedCluster_id}/apps/{app_id}/appAssets/{appAsset_id}`#

返回 JSON 响应正文中存储的 Application Asset API 资源。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| managedCluster_id | string | path | True  a| 
包含 managedCluster 资源的 ID



| app_id | string | path | True  a| 
包含应用程序资源的 ID



| appAsset_id | string | path | True  a| 
要列出的 appAsset 集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns the stored Application Asset API resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-appAsset"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| assetType | string | True  a| 
包含一个值的 JSON 字符串，该值指示从 Kubernetes 获取的资源类型。定义的值为：

* 最小长度：1
* 最大长度：254




| creationTimestamp | string | True  a| 
包含指示 Kubernetes 元数据创建时间的时间戳的 JSON 字符串。定义的值为：

* 符合 ISO-8601 日期时间方案




| GVK | link:#GVK["GVK"] | True  a| 
包含 Kubernetes 组/版本/种类的 JSON 对象。



| resource | link:#type_json["type_json"] | False  a| 
包含 Kubernetes 资源定义的全部内容的 JSON 对象。



| 资产ID | string | True  a| 
包含 Kubernetes 分配的标识符的 JSON 字符串。定义的值为：

* 符合 Astra 标识符架构




| labels | 数组[link:#labels["labels"]] | True  a| 
JSON 对象的 JSON 数组，每个都包含与资产关联的 Kubernetes 标签。如果没有可用的详细信息，此数组将为空。

* 最小项目数：0




| assetName | string | True  a| 
包含 Kubernetes 分配的名称的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：254




| namespace | string | False  a| 
包含 Kubernetes 分配的命名空间的 JSON 字符串。定义的值为：

* 最小长度：1
* 最大长度：254




| 元数据 | link:#type_astra_metadata["type_astra_metadata"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构
