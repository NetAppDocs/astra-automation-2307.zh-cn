= 检索功能
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-get]#GET# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/features/{feature_id}`#

返回 JSON 响应正文中存储的 Feature API 资源。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}




| feature_id | string | path | True  a| 
要列出的功能集合的 ID

|===


== 响应

[source]
----
Status: 200, Returns the stored Feature API resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-feature"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"
* "1.1"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| name | string | True  a| 
包含功能名称的 JSON 字符串，采用分层点表示法。定义的值为：

* 最小长度：1
* 最大长度：63
* 此字符串经过内部验证，以防止 XSS、Unicode、目录遍历和 SQL 注入攻击。




| isEnabled | string | True  a| 
JSON 字符串，包含指示功能标志是打开还是关闭的值。定义的值为：

* "true" - 功能标志已启用。
* "false" - 功能标记已禁用。




| 元数据 | link:#type_astra_metadata["type_astra_metadata"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构


|===

  "version": "1.1",

  "name": "astra.account.rbac",
  "isEnabled": "true",
  "metadata": {
    "labels": [],
    "creationTimestamp": "2022-10-06T20:58:16.305662Z",

    "createdBy": "8f84cf09-8036-51e4-b579-bd30cb07b269"
  }
}
----
====
