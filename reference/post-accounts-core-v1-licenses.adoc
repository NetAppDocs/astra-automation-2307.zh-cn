= 创建新许可证
:allow-uri-read: 


[.api-doc-operation .api-doc-operation-post]#POST# [.api-doc-code-block]#`/accounts/{account_id}/core/v1/licenses`#

指示要创建的 License API 资源的所需值。



== 参数

[cols="5*"]
|===
| 名称 | Type | 在 | Required | Description 


| account_id | string | path | True  a| 
包含帐户资源的 ID

* 示例：{{.Account}}


|===


== 请求正文

指示要创建的 License API 资源的所需值。

[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-license"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| 分配 | string | False  a| 
包含指示许可证分配对象的 ID 的 JSON 字符串。仅在分配许可证时存在。对于 Astra Control Center 许可证，此字段必须填写帐户 ID，并且必须与 POST URI 中指定的帐户 ID 匹配。定义的值为：

* 符合 Astra 标识符架构




| 设备凭证 ID | string | False  a| 
包含与凭据 API 关联的 deviceCredential 标识符的 JSON 字符串。仅适用于 Astra Data Store 许可证。定义的值为：

* 符合 Astra 标识符架构




| licenseText | string | True  a| 
包含从 NetApp 许可证系统获取的许可证 JSON 的 base64 编码表示的 JSON 字符串。必须以原始字节级别顺序提供以进行签名验证。定义的值为：

* 符合 base64 编码数据架构




| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | False  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。


|===
.请求示例
[%collapsible]
====
[source, json, subs="+macros"]
----
{
  "type": "application/astra-license",
  "version": "1.0",
  "allocation": "4185d09b-6687-4801-a832-c005a9d94f41",
  "licenseText": "ewogICAic3RhdHVzUmVzcCI6ewogMTYwNzAwIgp9"
}
----
====


== 响应

[source]
----
Status: 201, Returns the newly created license resource in the JSON response body.
----
[cols="4*"]
|===
| 名称 | Type | Required | Description 


| type | string | True  a| 
资源的媒体类型。定义的值为：

* "application/astra-license"




| version | string | True  a| 
资源的版本。定义的值为：

* "1.0"




| id | string | True  a| 
资源的全局唯一标识符。定义的值为：

* 符合 UUIDv4 架构




| 分配 | string | False  a| 
包含指示许可证分配对象的 ID 的 JSON 字符串。仅在分配许可证时存在。对于 Astra Control Center 许可证，此字段必须填写帐户 ID，并且必须与 POST URI 中指定的帐户 ID 匹配。定义的值为：

* 符合 Astra 标识符架构




| hostID | string | False  a| 
包含许可证绑定到的主机标识符的 JSON 字符串。仅在许可证处于主机锁定状态时存在。定义的值为：

* 最小长度：1
* 最大长度：63




| 设备凭证 ID | string | False  a| 
包含与凭据 API 关联的 deviceCredential 标识符的 JSON 字符串。仅适用于 Astra Data Store 许可证。定义的值为：

* 符合 Astra 标识符架构




| isEvaluation | string | True  a| 
表示许可证是否为评估许可证的 JSON 字符串。定义的值为：

* "true"
* "false"




| licenseProtocol | string | True  a| 
包含从许可证中提取的许可证协议的 JSON 字符串。定义的值为：

* 架构未验证的字符串内容




| licenseText | string | True  a| 
包含从 NetApp 许可证系统获取的许可证 JSON 的 base64 编码表示的 JSON 字符串。必须以原始字节级别顺序提供以进行签名验证。定义的值为：

* 符合 base64 编码数据架构




| validFromTimestamp | string | True  a| 
包含指示许可证何时生效的时间戳的 JSON 字符串。定义的值为：

* 符合 ISO-8601 日期时间方案




| validUntilTimestamp | string | True  a| 
包含指示许可证何时不再有效的时间戳的 JSON 字符串。定义的值为：

* 符合 ISO-8601 日期时间方案




| 产品 | string | True  a| 
包含从许可证中提取的产品字符串的 JSON 字符串。定义的值为：

* 架构未验证的字符串内容




| productVersion | string | True  a| 
包含从许可证中提取的版本字符串的 JSON 字符串。定义的值为：

* 架构未验证的字符串内容




| 产品 SN | string | True  a| 
包含从许可证中提取的序列号字符串的 JSON 字符串。定义的值为：

* 架构未验证的字符串内容




| features | string | True  a| 
包含从许可证中提取的包值的 JSON 字符串。定义的值为：

* 架构未验证的字符串内容




| 容量 | string | True  a| 
包含从许可证中提取的容量的 JSON 字符串。



| capacity2 | string | True  a| 
包含从许可证中提取的辅助容量的 JSON 字符串。



| 插件 | 数组[link:#addons["插件"]] | False  a| 
包含许可证中附加组件列表的 JSON 对象的 JSON 数组。附加组件是指在未来可能发生的许可证变更。



| 元数据 | link:#type_astra_metadata_update["type_astra_metadata_update"] | True  a| 
与资源关联的客户端和服务指定的元数据。定义的值为：

* 符合 Astra 元数据架构如果在创建时未指定，则将创建没有标签的元数据对象。如果在更新时未指定，则元数据对象的标签、creationTimestamp 和 createdBy 将保留而不进行修改。
